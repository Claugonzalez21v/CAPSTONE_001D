Como ya tienes acceso a PostgreSQL, vamos a comprobarlo directamente.

Ejecuta en PostgreSQL:

SELECT
    username,
    is_staff,
    is_superuser,
    is_active
FROM auth_user;

Si el usuario admin aparece con:

is_staff = f
is_superuser = f

entonces Django Admin lo bloqueará.

Solución

En PostgreSQL ejecuta:

UPDATE auth_user
SET is_staff = TRUE,
    is_superuser = TRUE
WHERE username = 'admin';

Luego verifica:

SELECT
    username,
    is_staff,
    is_superuser
FROM auth_user
WHERE username = 'admin';

Si tu tabla se llama reservas_reserva:

FROM reservas_reserva;
Ver reservas ordenadas por fecha de creación
SELECT *
FROM reservas_reserva
ORDER BY creada DESC;
Ver reservas con información importante
SELECT
    id,
    usuario_id,
    email,
    servicio,
    fecha,
    hora,
    precio,
    creada
FROM reservas_reserva
ORDER BY fecha;
Ver reservas de un usuario específico
SELECT *
FROM reservas_reserva
WHERE usuario_id = 1;
Ver cuántas reservas existen
SELECT COUNT(*)
FROM reservas_reserva;
Ver reservas junto al nombre del usuario
SELECT
    r.id,
    u.username,
    r.email,
    r.servicio,
    r.fecha,
    r.hora,
    r.precio
FROM reservas_reserva r
INNER JOIN auth_user u
ON r.usuario_id = u.id
ORDER BY r.fecha;
